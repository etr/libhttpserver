7. [x] **code-quality-reviewer** | `scripts/check-release-notes.sh:163` | code-readability
   The section-heading grep pattern `^##[ \t]+.*${sec}` uses `[ \t]` inside a `-E` (ERE) context. POSIX ERE does not define `\t` as a character class shorthand; `\t` inside `[...]` is implementation-defined. On GNU grep and BSD grep it happens to work (both interpret `\t` as a tab in character classes), but it is non-portable. The same pattern appears in the awk-based `extract_section_body` function (line 234) where `\t` inside a character class is equally non-portable in POSIX awk.
   *Recommendation:* Use `[[:space:]]` instead of `[ \t]` for POSIX portability, e.g. `^##[[:space:]]+.*${sec}`. Alternatively, use a literal tab via `$'\t'` if a tab is specifically intended and a space should not match.
   *Fixed:* Replaced all `[ \t]` with `[[:space:]]` in grep and awk patterns throughout the script.
