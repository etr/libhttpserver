9. [x] **security-reviewer** | `examples/centralized_authentication.cpp:65` | cryptographic
   constant_time_equal uses an XOR-accumulate loop without a volatile accumulator. A sufficiently aggressive compiler could theoretically short-circuit the loop once diff becomes non-zero, since subsequent iterations cannot change the outcome. In practice GCC and Clang do not perform this optimization (the loop body has memory reads with potential side effects), but the C++ abstract machine does not prohibit it. The safer form is: volatile unsigned char diff = 0; Alternatively, use CRYPTO_memcmp or timingsafe_bcmp from the TLS library already linked.
   *Recommendation:* Declare the accumulator as 'volatile unsigned char diff = 0;' to prevent the compiler from proving the loop is redundant and eliding iterations. Both centralized_authentication.cpp and per_route_auth.cpp share this identical implementation and should be updated together.
   *Worked 2026-07-12 (chore/unworked-minor-issues):* Added 'volatile' to the 'unsigned char diff = 0;' accumulator in examples/centralized_authentication.cpp's constant_time_equal, matching the already-fixed sibling in per_route_auth.cpp.
