1. [x] **architecture-alignment-checker** | `src/httpserver/webserver.hpp:504` | interface-contract
   register_ws_resource takes a raw websocket_handler* pointer. §4.1 (webserver component spec) and DR-010 both specify the signature as register_ws_resource(path, unique_ptr<websocket_handler>) plus a shared_ptr overload. The raw-pointer form was present in the feature/v2.0 base and was carried forward unchanged by this task. This task's stated scope was only removing the #ifdef HAVE_WEBSOCKET guard, but the raw-pointer contract violation now surfaces in the unconditionally-visible public API and is introduced to consumers who previously never saw this method on HAVE_WEBSOCKET-off builds.
   *Recommendation:* Add unique_ptr<websocket_handler> and shared_ptr<websocket_handler> overloads as specified in §4.1 and DR-010. The raw-pointer overload can be removed or kept as a deprecated shim. DR-010 explicitly states: 'Raw-pointer overload removed (extending PRD-HDL-REQ-005).'
   **Status:** Already addressed by TASK-035 — webserver_websocket.hpp has unique_ptr<T> template + shared_ptr overloads; raw pointer removed. Verified 2026-05-27.

2. [x] **code-quality-reviewer** | `src/webserver.cpp:253` | test-coverage
   The webserver constructor guards HAVE_BAUTH (basic_auth_enabled throws feature_unavailable) and HAVE_GNUTLS (use_ssl throws feature_unavailable) but has no matching guard for HAVE_DAUTH. Enabling digest_auth on a HAVE_DAUTH-off build silently constructs the server instead of throwing feature_unavailable. The task spec (§7) says every build-disabled feature invoked at construction time should throw. The tests in create_webserver_test.cpp cover the TLS and basic_auth throw paths but there is no corresponding #ifndef HAVE_DAUTH guard and no test for digest_auth_true on a HAVE_DAUTH-off build.
   *Recommendation:* Add `#ifndef HAVE_DAUTH if (digest_auth_enabled) { throw feature_unavailable("digest_auth", "HAVE_DAUTH"); } #endif` in the webserver constructor body alongside the HAVE_BAUTH and HAVE_GNUTLS checks, and add a parallel `#ifndef HAVE_DAUTH` test case in create_webserver_test.cpp covering the throw and the false-is-ok paths.
   **Status:** Already addressed — guard exists at webserver.cpp lines 258-266; tests exist in webserver_dauth_unavailable_test.cpp. Verified 2026-05-27.
