23. [ ] **security-reviewer** | `src/http_request.cpp:492` | insecure-design
   In populate_all_cert_fields(), client_cert_fields_cached is set to true (line 492) before any cert field is populated (lines 510-581). If std::bad_alloc is thrown during the two-pass DN/CN string allocation (e.g., std::string dn(dn_size, '\0')), the cached flag is already true. Subsequent calls return immediately and all string_view accessors return the empty default sentinels — which is the correct sentinel value.    *Recommendation:* Consider moving client_cert_fields_cached = true to after all field population is complete (or after the early-exit for !cert.is_valid()), and restructure populate_all_cert_fields() so that a bad_alloc on one field does not prevent population of subsequent fields (e.g., catch per-field allocation failures individually).
