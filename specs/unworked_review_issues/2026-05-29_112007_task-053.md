## Minor — Deferred (substantive)

### Cluster 1: `matched_path_template` carries `standardized_url`, not the registered pattern (#7, #9, #10, #28, #34)

On cache hit (and on regex/radix tier-miss) `mr->matched_path_template` is the concrete request URL, not the registered pattern (e.g. `/users/42` instead of `/users/{id}`). The hook contract for `route_resolved_ctx::matched->path_template` implies a registered pattern. Right fix: add a `std::string path_template` to `cache_value`, populate at insert, restore at hit. Cost: ~24-32 bytes per cache entry (256 entries → ~6-8 KB) — negligible. Also documents the CWE-116 confused-deputy concern in the hook_context headers.

### Cluster 2: any_hooks_ memory ordering (#13, #30)

The class-level doc-comment on `any_hooks_` prescribes `memory_order_acquire` but `has_hooks_for()` uses `memory_order_relaxed`. In practice safe (firing sites acquire `hook_table_mutex_` via `shared_lock`), but TSan on ARM64 will flag it. Two valid fixes: (a) document that the mutex acquire provides the happens-before edge, accepting the relaxed gate, or (b) switch to acquire to match the prescription. Pre-existing inconsistency, not introduced by TASK-053.

### Cluster 3: dispatch hot-path allocation (#25, #26, #27)

`canonicalize_lookup_path` allocates unconditionally even for already-canonical paths. TASK-058 step 1 has already converted the return type to `string_view`; the remaining items are (a) avoid the `cache_key` allocation on the canonical-path hit path and (b) move (not copy) `captured_params` into `cache_value` on the miss path. Both are substantive — they thread lifetime/move semantics through the resolver and cache.

### Cluster 4: route_cache double-probe + contention (#21, #29)

`find_by_view` does a manual bucket walk and then a second `map_.find(...)` to get a mutable iterator for the splice. The double probe is avoidable by typing the bucket-walk loop on the non-const overload. Separately, the full `cache_value` copy under the exclusive mutex is the contention bottleneck on parameterized routes — deferring the copy to after the lock requires storing the value in a separately heap-allocated node (e.g. `shared_ptr<cache_value>`).

### Cluster 5: v2_dispatch_contract test infrastructure (#15, #23, #45, #46, #49, #50)

A test-infra pass: replace the per-test `sleep_for(50ms)` with `wait_for_server_ready()` (already partially done — verify), factor `make_probe_hook(hook_probe&)` to deduplicate the four near-identical lambdas, add inter-test port-reuse spacing, and document the `ws.stop()` happens-before edge that makes the relaxed atomic reads after it safe. Optionally add `curl_global_init`/`cleanup` at the env level.
