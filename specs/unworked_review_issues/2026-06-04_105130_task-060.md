1. [x] **code-simplifier** | `scripts/check-warning-suppressions.sh:61` | code-structure
   Two separate awk invocations are launched per candidate pragma line: one scans the entire file for the nearest push before the target line, and a second scans the entire file for the nearest pop after. For each matched line this means two full file passes. A single awk invocation can record push/pop line numbers and decide the result in one pass, cutting the work in half and keeping the logic in one place.
   *Recommendation:* Combine both checks into one awk call. Example:

  bracketed=$(awk -v target="$lineno" '
    /^#pragma[[:space:]]+GCC[[:space:]]+diagnostic[[:space:]]+push/ && NR < target { last_push = NR }
    /^#pragma[[:space:]]+GCC[[:space:]]+diagnostic[[:space:]]+pop/  && NR > target && !first_pop { first_pop = NR }
    END { print (last_push && first_pop) ? "yes" : "no" }
  ' "$file")
  if [ "$bracketed" = "no" ]; then
      echo "  $file:$lineno: file-scoped ..." >&2
      violations=$((violations + 1))
  fi

This removes the two separate shell variables push_before/pop_after and the two-condition OR test, replacing them with a single descriptive boolean result.
