Classes, relationships & filesystem — webserver and its collaborators

Post-DR-014: webserver is a thin façade over webserver_impl, which is a pure composition root holding 5 state collaborators (own mutex + data) and 7 behavior services (stateless, constructor-injected by const&), plus a static MHD adapter facet. Colour encodes the role; every card carries its header and translation units. Paths are relative to src/.

01

Composition backbone

Ownership is strictly linear: create_webserver builds a webserver, which owns one webserver_impl (unique_ptr), which holds all twelve collaborators by value. Services below reference the state collaborators and each other by const&; construction order (state → leaf services → dispatcher → pipeline) is deliberate and enforced by -Wreorder. Every service also holds const webserver_config& (reached via parent->config) — shown as a dashed cfg chip.

▪ MHD adapter facet — static C-ABI trampolines on webserver_impl (not an 8th service)

libmicrohttpd calls these by function-pointer address. Each casts cls → webserver → impl_, marshals the C ABI (sockaddr → peer_address, con_cls/socket_context lifetime), and delegates all behavior to a service. They must live on the impl to reach every collaborator — the trampoline is the adapter.

answer_to_connection ▸pipeline_ connection_notify ▸hooks_dispatch_ policy_callback ▸acl_ request_completed ▸hooks_dispatch_ post_iterator ▸upload_ uri_log error_log unescaper_func psk / sni_cert GnuTLS

All trampolines live in one adapter TU: detail/webserver_callbacks.cpp

Translation-unit map — one-class-per-file shape: webserver = a single façade TU (webserver.cpp, 552 SLOC); webserver_impl = 2 TUs (composition root + MHD adapter, 108 + 381 SLOC). Both webserver.cpp and webserver_callbacks.cpp sit under the 600-SLOC façade/adapter carve-out (other files: 500) — check-file-size gate PASSES. No file spans both classes. Reading the graph. Solid ownership (◆) flows one way and top-down: webserver → webserver_impl → collaborators (by value) → their owned value types. Behavior services never own state — they hold const& (▸) into the state collaborators and each other, wired once in the webserver_impl constructor, forming an acyclic DAG with no back-pointer (the sole exception is daemon_lifecycle, which needs webserver_impl* to read broad config while building the MHD option array). The static MHD trampolines are the only code that touches libmicrohttpd's C ABI. Header convention: public surface under src/httpserver/, internal detail headers under src/httpserver/detail/ (never installed), implementations under src/ and src/detail/.
